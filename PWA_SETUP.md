# 404 Found - PWA Setup Guide

## Installation

```bash
npm install
npm run build
```

## PWA Features

### Stale-While-Revalidate Strategy
The app uses Workbox with the following caching strategies:

1. **Static Assets** (JS, CSS, images, fonts)
   - Strategy: `StaleWhileRevalidate`
   - Cache: 30 days
   - Max entries: 100

2. **Next.js Static Chunks** (`/_next/static/*`)
   - Strategy: `StaleWhileRevalidate`
   - Cache: 30 days
   - Max entries: 50

3. **API Calls** (`/api/*`)
   - Strategy: `NetworkFirst`
   - Timeout: 3 seconds
   - Cache: 5 minutes

4. **HTML Pages**
   - Strategy: `StaleWhileRevalidate`
   - Cache: 1 day
   - Max entries: 20

### Offline Support
- Fallback page: `/offline.html`
- Service worker auto-registers
- Works with zero connectivity
- Cached data persists across sessions

## Theme Colors

The PWA uses the tactical terminal theme:
- **Background**: `#0D0D19` (deep purple/slate)
- **Theme**: `#0D0D19`
- **Accent**: `#9333ea` (purple)
- **Alert**: `#ff0040` (red)
- **Success**: `#00ff41` (green)

## Testing PWA

1. **Build the app**:
   ```bash
   npm run build
   ```

2. **Start production server**:
   ```bash
   npm start
   ```

3. **Install as PWA**:
   - Chrome: Look for install icon in address bar
   - Safari (iOS): Share → Add to Home Screen
   - Edge: Settings → Apps → Install this site as an app

4. **Test offline**:
   - Open DevTools → Network → Set to "Offline"
   - Refresh page - should show cached content
   - Click "Attempt Reconnect" when online again

## Service Worker

The service worker is automatically generated by `next-pwa` during build:
- Output: `public/sw.js`
- Workbox version: Latest
- Precache: All static assets
- Runtime cache: Dynamic content

## Manifest

Location: `public/manifest.json`

Includes:
- App name and description
- Icons (72px - 512px)
- Theme colors
- Shortcuts (New Alert, Scan QR)
- Screenshots for install prompts

## Browser Support

- Chrome/Edge: Full PWA support
- Safari (iOS): Limited support (no background sync)
- Firefox: Basic support

## Notes

- Service worker is disabled in development mode
- First load requires network to cache assets
- Subsequent loads work offline
- API calls timeout after 3 seconds when offline
